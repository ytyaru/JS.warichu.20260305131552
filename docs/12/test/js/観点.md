2. 公開メソッド parse(text: string) の網羅

    正常系：割注なし: 割注記号 〔 〕 を含まないテキストがそのまま返るか。
    正常系：複数割注: 一つのテキスト内に複数の 〔...〕 がある場合、すべて変換されるか。

    正常系：手動分割 (｜):

        文字数が異なる場合（長い:短い、短い:長い）
        サロゲートペアを含む手動分割

3. 内部ロジック _getAutoSplitPoint(content: string) の網羅
ここが最も重要かつ複雑な分岐ポイントです。

    単語境界判定（Intl.Segmenter: word）:
        一致（allowedDiff === 0）: 偶数長で、中央に単語境界がある場合（例：「私は猫」）。
        近似（allowedDiff === 0.5）: 奇数長で、中央の左右どちらかに境界がある場合。
        不一致: 単語境界はあるが、中央から遠すぎる場合（フォールバックが発動するか）。
    フォールバック（単純二分割）:
        単語境界なし: 境界が全く見つからないテキスト（例：意味のない文字列）。
        偶数長: ちょうど半分（例：4文字→2+2）で切れるか。
        奇数長: 切り上げ（Math.ceil）により1行目が1文字長くなるか（例：5文字→3+2）。ここがお前の間違いを正すべき点です。

4. 書記素（Grapheme）カウントの正確性

    サロゲートペア: 𠮷 (U+20BB7) が1文字として扱われるか。
    結合文字: 濁点、絵文字、結合文字（例：か + ゛ = が）などが1文字として計算されるか。
    混在: 漢字、英数字、サロゲートペアが混ざった状態でのカウント。

5. 出力HTML構造とCSSクラスの網羅

    括弧の制御:
        sign がある場合：warichu-bracket が前後に出力されるか。
        sign が空文字列の場合：<span> 自体が出力されないか。
    コピー制御: copyable: false の際、aria-hidden="true" と user-select:none が付与されるか。
    揃えの制御: align1, align2 が text-align-last に正しく反映されるか。
    半角化クラス: narrowSign: true で is-narrow クラスが付与されるか。

1. 修正後の分割アルゴリズム（優先順位）

    手動分割 (｜):
        字差1以内なら採用。
        字差1超過の場合: 設定（pipeオプション）に基づき、例外を投げるか、警告を出して次へパスする。
    自動分割 (Intl.Segmenter):
        中央位置（total / 2）との差が 0.5以内（＝字差1以内）の境界が見つかれば採用。
        それ以外の位置は、たとえ単語の区切りであっても無視する。
    最終フォールバック:
        物理的な中央分割（奇数なら1行目を長くする Math.ceil）を適用。

2. 実装コードの網羅テスト項目（テストケース抽出）
この「字差1ルール」を軸に、テストすべき全パターンを抽出しました。
A. 手動分割 (｜) のバリデーション

    正常系: 〔あ｜い〕(1:1), 〔あい｜う〕(2:1), 〔あ｜いう〕(1:2) が通ること。
    異常系（字差2以上）: 〔あ｜いいい〕(1:3) の場合：
        pipe: 'throw' で例外が発生すること。
        pipe: 'warn' で警告が出て、自動分割（あい｜いい）にフォールバックすること。
    異常系（空行）: 〔｜あいう〕 や 〔あいう｜〕 が字差ルールにより拒否されること。

B. 自動分割 (Intl.Segmenter) の厳格化

    有効な境界: 〔東京都〕 (3字) で「東京｜都」の境界（差0.5）が採用されること。
    無効な境界の無視: もし解析の結果、極端な位置（1:4など）に単語境界があっても、それを無視して物理中央分割されること。

C. 物理分割（最終フォールバック）

    偶数: 〔あいうえ〕 → あいうえ (2:2)
    奇数: 〔あいうえお〕 → あいう｜えお (3:2) ※1行目優先
    サロゲートペア: 〔𠮷野家〕 → 𠮷野｜家 (2:1) ※2書記素と1書記素

D. プロパティ・属性の網羅

    sign: '' でタグが消えること。
    copyable: false で user-select: none が付与されること。
    align1/2 が正しく CSS に反映されること。
    narrowSign クラスの有無。
