# 処理フローの一般化とビルド戦略

本ドキュメントは[fallback.md][]における処理を一般化したコードの実装とビルドについて考察する。

[fallback.md]:./fallback.md

処理フローにはフォールバック以外にもパターンがある。それらを異なる関数としてAPI化し提供するライブラリを実装すれば、責任分離した保守性の高いコードを書けるようになる。これを目指す。

この際、プロジェクトにおいて使用しないコードもあるだろう。ライブラリ使用者が無駄なく必要なコードだけをビルドするツールの実装まで考察した。

尚、本文書のコード内容は、割注においては冗長であるため実装しない。あくまで考察と概要コードのメモに留める。
